# BERT-Heavy + LLM-Final アーキテクチャにおける新規性方向性 検討ノート

> 作成日: 2026年6月3日
> 背景: 現行アーキテクチャ（LLM→アスペクト抽出 + BERT→感情分類）の処理速度ボトルネックを解消するため、BERTがメイン処理を担い、LLMを最終ステップ（自然言語化・分析）のみに限定する設計への移行を検討する。

---

## 現行アーキテクチャとボトルネック

```
[現行]
口コミ文
  ↓
LLM (Qwen3 4B) ← ★ ここがボトルネック（重い）
  アスペクト抽出 + 根拠文抽出
  ↓
BERT (cl-tohoku + koheiduck finetuned)
  感情分類 (Positive / Neutral / Negative)
  ↓
アスペクト別感情スコア
  Accuracy: 0.900
```

**問題**: LLMが全口コミを1文ずつ処理するためスループットが低く、大量の口コミを扱うリアルタイム処理・定期バッチ処理の両方で実用上の障壁となる。

---

## 目指すアーキテクチャ

```
[新設計]
口コミ文
  ↓
BERT (主処理担当 — 高速・軽量)
  アスペクト検出 + 感情分類
  ↓
構造化アスペクト感情スコア (JSON/CSV)
  ↓
LLM (最終ステップのみ — 少量・低コスト)
  自然言語化 / 分析 / 推薦理由生成
  ↓
出力: 価格推薦テキスト / インサイト / ペルソナ記述
```

## 調査で確認された先行研究の状況

### 確認された類似アーキテクチャの傾向（2023〜2026）

| 論文・文献 | アーキテクチャの特徴 | 調査確認状況 |
|-----------|-------------------|------------|
| Hellwig et al. (2026) *arXiv:2603.01778* LA-ABSA | LLMでアノテーション生成 → 軽量BERTをfine-tune → 推論はBERT-only | ★ 直接関連（LREC 2026採録） |
| Chatterjee et al. (2026) *hal.science* | 「LLM-in-the-loop」: HAN(効率的なSystem 1) + LLM(System 2) の役割分担 | ★ 直接関連 |
| Marutho & Rustad (2024) *J. Open Innovation* | Sentence-BERT embedding + ベイズ探索クラスタリング + スパース注意機構によるABSA | 類似方向 |
| Polpinij et al. (2025) *IEEE* | ハイブリッドembeddingによるホテルレビューの暗黙的アスペクト・感情検出 | 類似方向 |
| Chaudhary & Neal (2026) *ACM Computing Surveys* | 暗黙的アスペクト抽出の体系的サーベイ: BERT embedding + コサイン類似度が主流 | サーベイ |

**重要な観察**: 「LLMでアノテーション → BERTをfine-tuning → 推論はBERT-only」という設計（LA-ABSAパターン）は2026年に登場しつつあるが、**日本語・ホテルドメインへの適用**は存在しない。

## 新規性のある研究方向性（4方向）

---

### N1. LLM-as-Annotator → 日本語ホテル特化BERT Fine-tuning（＋ Continual Learning）

#### コンセプト

LLMは「一度だけ教師データを作るアノテーター」として使用し、以後の推論はBERT単独で行う。さらに、口コミが定期的に蓄積されるたびにBERTを自動更新するオンライン学習ループを加える。

```
[初回セットアップ（一度だけLLM使用）]
既存口コミ corpus
  → LLM: アスペクト＋根拠文＋感情をアノテーション (LA-ABSAパターン)
  → 生成されたアノテーションでBERTをfine-tune
  → ホテル日本語ドメイン特化ABSAモデル完成

[推論フェーズ（LLM使用なし）]
新口コミ → BERT → アスペクト別感情スコア (高速)

[自動更新ループ（Continual Learning）]
月N件の口コミが蓄積 → 自動的にLLMでアノテーション追加
  → BERTを漸進的に再fine-tune（Elastic Weight Consolidation等で忘却防止）
  → 時代・施設による口コミ表現変化に自動適応
```

#### 新規性のポイント

- **LA-ABSAパターン（Hellwig et al. 2026）との差別化**: 日本語・ホテルドメイン特化 + Continual Learning の組み合わせ
- **技術的貢献**: 月5〜10件程度の疎データ環境（S-1課題）でも機能するオンライン更新設計
- **実務的貢献**: 地方宿泊施設が所持するドメイン特化モデルを低コストで維持できるシステム

#### 技術的課題と研究設問

- 少量LLMアノテーション（N=50〜200）でBERTがどこまで精度を維持できるか → 低リソース評価
- Continual Learningによる忘却（Catastrophic Forgetting）の防止戦略
- 既存精度（Accuracy 0.900）から更に改善できるか vs. LLM推論コスト削減のトレードオフ定量化

---

### N2. BERT Embedding × コサイン類似度による暗黙的アスペクト自動検出（日本語特化）

#### コンセプト

日本語口コミに特有の**暗黙的アスペクト表現**（「また来たい」→ 再訪意向、「価格の割に」→ コスパ）をLLMなしに検出するため、BERTembeddingとアスペクト記述のコサイン類似度マッピングを活用する。

```
[アスペクトプロトタイプの構築（オフライン・一度だけ）]
各アスペクト（食事・立地・清掃など10カテゴリ）の
  代表的な表現文を複数収集 → BERT embeddingで平均ベクトルを計算
  → アスペクト別プロトタイプベクトル群を構築

[推論フェーズ（完全BERT-only）]
口コミ文を文単位に分割
  → Sentence-BERT でembedding
  → 各文とアスペクトプロトタイプのコサイン類似度を計算
  → 最近傍アスペクトに自動割り当て（閾値以下は「非該当」）
  → 割り当てられた文をBERT感情分類器へ入力
  → アスペクト別感情スコア生成

[LLMは後処理のみ]
低信頼度（コサイン類似度が全アスペクトに対して低い）の文のみ
  → LLMに問い合わせ（コスト最小化）
```

#### 新規性のポイント

- **暗黙的アスペクト（Implicit Aspect）への対応**: 現行システムはLLMが担っていた暗黙表現の処理を、BERTembeddingの意味空間で代替
- **日本語特化の新規性**: 英語のImplicit Aspect研究（Chaudhary & Neal, 2026）はあるが、日本語ホテル口コミでの体系的研究は存在しない
- **LLMコール最小化設計**: 低信頼度ケースのみLLMにフォールバックする「Selective LLM Routing」

#### 技術的課題と研究設問

- 日本語Sentence-BERTモデル（`sonoisa/sentence-bert-base-ja-mean-tokens` 等）のアスペクト識別精度
- 閾値設計（コサイン類似度の分布から適応的に決定する方法）
- 「婉曲表現」「二重否定」など日本語特有の難事例に対するBERTembeddingの限界分析

---

### N3. データドリブン・アスペクト自動発見パイプライン（BERT Clustering + LLM命名）

#### コンセプト

固定の10アスペクトカテゴリを事前に人手で定義するのではなく、**口コミデータから帰納的にアスペクトを自動発見**する。LLMはクラスタに「名前を付ける」最終ステップのみに限定する。

```
[アスペクト自動発見（データドリブン）]
大量ホテル口コミ corpus
  → BERT/Sentence-BERT で文embedding化
  → k-means or HDBSCAN でクラスタリング
       （エルボー法/シルエット係数でk決定）
  → [LLM、ここで1回だけ使用]
       各クラスタの代表文群 → LLMが「アスペクトカテゴリ名」を命名
  → 施設・地域・時代に応じた動的アスペクト体系が完成

[アスペクト体系を使った感情分析（以後LLM不要）]
新口コミ → N2の方法（コサイン類似度）でアスペクト割り当て
  → BERT感情分類
  → スコア生成
```

#### 新規性のポイント

- **現行研究との差別化**: Degife & Lin (2024) では人手で9グループにまとめていた。本方向性は完全自動化
- **時系列適応性**: コロナ対策・インバウンド対応・SDGs等、時代によって登場する新アスペクトを自動検出できる
- **施設タイプ適応**: 旅館・ビジネスホテル・ゲストハウスで異なるアスペクト体系が自然に形成される
- **昨日調査のペルソナ研究との接続**: BERT Clustering + LLM命名パターン（Shin et al. 2024等）のABSAへの応用

#### 技術的課題と研究設問

- 最適なクラスタ数kの決定方法（施設タイプや口コミ量によって変わる）
- LLMが付与したアスペクト名の妥当性評価方法（ドメイン専門家との一致率）
- 施設間でアスペクト体系がどの程度異なるか（汎用性 vs. 特化性のトレードオフ）

### N4. BERT-ABSA → LLM-Based Dynamic Pricing Explanation Generator

#### コンセプト（最も実装研究として完結性が高い）

BERTがすべてのABSA処理を行い、LLMは**価格推薦の自然言語説明を生成する最終ステップのみ**に使用する。「なぜこの価格を推薦するのか」をABSAスコアの根拠に基づいてLLMが生成することで、ホテル運営者の信頼と受容性を高める説明可能DP（XDP）を実現する。

```
[処理パイプライン全体]
口コミ corpus
  ↓ (高速・バッチ処理)
BERT-only ABSA [N1 or N2の手法]
  → 10アスペクト × 感情比率 × 時系列 の構造化スコア

  ↓ (既存RD-01〜RD-07のDP ロジック)
需要予測モデル (GRU / LightGBM 等)
  → 需要予測値・稼働率予測値

  ↓ (最適化)
価格最適化 (線形計画 / RL 等)
  → 推薦価格 P_rec

  ↓ [LLM、ここのみ]
LLM への入力:
  - P_rec（推薦価格）
  - 主要な根拠: アスペクトスコアの変化・競合比較・需要予測値
  - 前回価格との差分
  ↓
LLM の出力（価格推薦テキスト）:
  「先週と比べて清掃に関する口コミが改善した（+0.15）こと、
   立地アスペクトの評価が競合A施設を上回っていること（+0.22）を踏まえ、
   今週末の価格を13,500円（先週比+500円）に引き上げることを推薦します。」
```

#### 新規性のポイント

- **完全に未研究の組み合わせ**: ABSA構造化スコア → LLM価格推薦テキスト生成の組み合わせは確認された論文ゼロ
- **RD-03（説明可能DP）との直接対応**: `local_dp_app_rd_ideas.md` の課題T-1（RMSのブラックボックス性）、P-2（RMSへの不信）を技術的に解決
- **LLMコスト最小化**: LLMへの入力は「要約済みスコアと推薦値」のみ（短い構造化テキスト）。口コミ全文を処理させない
- **説明根拠の根拠付け（Groundedness）**: LLMが生成する説明はABSAスコアという数値的根拠に紐付いており、ハルシネーション耐性が高い

#### 技術的課題と研究設問

- LLMへのプロンプト設計: スコアをどのように要約・提示するか（表形式 vs. 自然言語 vs. 構造化JSON）
- 生成された説明文の評価方法: ホテル運営者へのユーザースタディ（受容性・理解度・信頼度）
- 説明文の根拠忠実性（Faithfulness）の定量評価: 生成テキストが実際のスコア根拠と整合しているか
