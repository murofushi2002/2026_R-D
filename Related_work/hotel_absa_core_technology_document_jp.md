# ホテル口コミABSA（Aspect-Based Sentiment Analysis）を用いた
# コア技術ドキュメント

## 1. 概要

本ドキュメントでは、ホテル口コミデータを対象とした Aspect-Based Sentiment Analysis（ABSA：観点別感情分析）のコア技術について整理する。
本研究では、ホテル口コミから「清掃」「食事」「部屋」「スタッフ」などの観点（Aspect）ごとの感情を抽出し、定量化した感情情報を需要予測および価格最適化へ活用することを目的とする。

特に本資料では、以下の技術領域に焦点を当てる。

- ホテル口コミABSAの技術背景
- 提案アーキテクチャ（LLM × BERT）
- 日本語口コミ解析における課題
- アスペクト抽出技術
- 感情分類技術
- ABSAデータセット構築方法
- 評価方法と実験設計

本資料は「ホテル口コミからABSA用データセットを作成する工程」までを対象範囲とする。

# 2. 研究背景

## 2.1 ホテル業界における課題

近年、ホテル業界では以下の課題が顕在化している。

- インバウンド需要回復による需要変動の増加
- Dynamic Pricing（動的価格設定）の普及
- 競合ホテルとの差別化困難
- 口コミ評価が売上へ与える影響の増加
- 顧客満足度と利益最大化の両立

従来のホテル運営では、レビューサイト上の数値スコア（4.2 / 5.0 など）が主要指標として利用されてきた。
しかし、数値スコアのみでは以下の問題が存在する。

| 問題 | 内容 |
|---|---|
| 解釈困難 | なぜ高評価/低評価なのか分からない |
| 粒度不足 | 「部屋」評価のどこが良いのか不明 |
| 意思決定困難 | 改善施策へ結び付けにくい |
| 時系列利用困難 | 需要予測へ組み込みにくい |

そのため、口コミテキストを構造化し、「どの観点に対して」「どの感情を持っているか」を抽出するABSAが重要となる。

# 3. Aspect-Based Sentiment Analysis（ABSA）

## 3.1 ABSAとは

ABSA（Aspect-Based Sentiment Analysis）は、文章全体ではなく「観点単位」で感情を分析する技術である。

通常の感情分析では、口コミ全体に対して Positive / Negative を付与する。

例：

「部屋はきれいだったが、食事は微妙だった」

通常感情分析：
- Positive または Neutral

ABSA：

| Aspect | Sentiment |
|---|---|
| 部屋・清掃 | Positive |
| 食事 | Negative |

このように、1つの口コミ文に複数の感情が混在していても解析可能となる。

---

## 3.2 ホテル口コミABSAの特徴

ホテル口コミは以下の特徴を持つ。

| 特徴 | 内容 |
|---|---|
| 多観点性 | 食事・立地・清掃など複数観点が存在 |
| 日本語特有表現 | 婉曲表現・否定・丁寧語が多い |
| 暗黙的感情 | 明示的な感情語を含まない |
| 複文構造 | 1文内に複数観点が混在 |
| 主観性 | 個人差が大きい |

例：

「立地は便利でしたが、部屋の匂いが気になりました。」

この場合：

| Aspect | Sentiment |
|---|---|
| 立地 | Positive |
| 部屋（匂い） | Negative |

となる。

そのため、単純な辞書ベース手法では十分な精度を得ることが難しい。

---

# 4. 提案手法の全体構成

## 4.1 システム概要

本研究では、LLMとBERTを組み合わせたハイブリッド型ABSAモデルを提案する。

処理フローは以下の通りである。

1. 口コミ文入力
2. LLMによる観点抽出
3. 根拠文抽出
4. BERTによる感情分類
5. 観点別感情スコア生成
6. データベース化

---

## 4.2 モデル構成

### Step1：LLMによるアスペクト抽出

LLMを用いて、口コミから対象観点を抽出する。

例：

入力：

「部屋はきれいだが、食事は微妙だった」

出力：

| Aspect | 理由文 |
|---|---|
| 清掃 | 部屋はきれい |
| 食事 | 食事は微妙 |

LLMは複数観点抽出と理由文抽出を担当する。

---

### Step2：BERTによる感情分類

抽出された理由文をBERTへ入力し、感情分類を行う。

| 入力 | 出力 |
|---|---|
| 部屋はきれい | Positive |
| 食事は微妙 | Negative |

BERTを利用することで、日本語特有の文脈依存表現への対応を強化する。

---

# 5. なぜLLM × BERTなのか

## 5.1 LLM単体の問題

LLMのみで感情分析を行う場合、以下の問題が存在した。

| 問題 | 内容 |
|---|---|
| 出力揺れ | 同一入力でも結果が変化 |
| 日本語否定表現 | 誤判定が発生 |
| 長文処理 | 複数観点で混乱 |
| 境界曖昧性 | 食事/立地などの混同 |

例：

「部屋はきれいとは言えませんでした」

この場合、LLM単体では「きれい」をPositiveとして誤判定するケースがあった。

---

## 5.2 BERT併用の利点

BERTは文脈理解能力に優れるため、否定表現や婉曲表現への対応性能が高い。

例：

| 文 | 正解 |
|---|---|
| きれいではない | Negative |
| 微妙だった | Negative |
| 悪くはない | Neutral |

そのため、

- LLM：観点抽出
- BERT：感情分類

という役割分担を行うことで精度向上を実現した。

# 7. ホテル口コミABSAにおけるアスペクト設計

## 7.1 アスペクト定義

本研究では、ホテル口コミを以下の観点へ分類する。

| Aspect | 内容 |
|---|---|
| 食事 | 朝食・夕食・味・量 |
| 立地 | アクセス・周辺環境 |
| スタッフ | 接客・対応 |
| 清掃 | 清潔さ・衛生 |
| 価格 | コスト感 |
| 部屋 | 広さ・快適性 |
| 雰囲気 | 高級感・内装 |
| 施設・設備 | 温泉・WiFi・設備 |
| 予約・手続き | チェックイン等 |
| 再訪意向 | また泊まりたいか |

---

## 7.2 アスペクト境界問題

ホテル口コミでは、アスペクト間境界が曖昧である。

例：

「外食できる店がたくさんあった」

候補：

- 食事
- 立地

この場合、本研究では「周辺施設」に関する記述として「立地」に分類する。

このような境界曖昧性がABSAの主要課題である。

# 8. プロンプト設計

## 8.1 プロンプト改善の必要性

LLMによるアスペクト抽出精度は、プロンプト設計へ大きく依存する。

本研究では、3段階のプロンプト改善を行った。

---

## 8.2 STEP1：簡易プロンプト

### 内容

「以下の口コミを観点別に分類してください」

### 問題

- 境界曖昧性
- 誤分類増加
- 根拠抽出不安定

### 精度

Accuracy = 0.813

---

## 8.3 STEP2：厳密プロンプト

### 内容

各アスペクトに対して詳細定義を付与。

例：

- 食事：ホテル内の朝食・夕食
- 周辺レストランは対象外

### 問題

制約が厳しすぎたため：

- 未抽出増加
- 過剰慎重化
- Recall低下

### 精度

Accuracy = 0.687

---

## 8.4 STEP3：学習過程に沿ったプロンプト

### 内容

LLMが自然に理解しやすい形で定義。

例：

- 食事：朝食・夕食
- 清掃：部屋のきれいさ

### 効果

- 抽出漏れ減少
- 境界理解向上
- 安定性向上

### 精度

Accuracy = 0.900

# 11. データ前処理

## 11.1 前処理の目的

口コミ文にはノイズが多く含まれるため、前処理を実施する。

---

## 11.2 実施内容

| 前処理 | 内容 |
|---|---|
| 重複除去 | 同一口コミ除去 |
| 記号除去 | 不要文字除去 |
| Unicode正規化 | 全角半角統一 |
| 改行整形 | 文構造整理 |
| 絵文字除去 | ノイズ削減 |
| URL除去 | 外部リンク除去 |

---

## 11.3 文分割

複数観点を含む文章を分割する。

例：

入力：

「部屋はきれいだったが、食事は微妙だった」

分割後：

- 部屋はきれいだった
- 食事は微妙だった

これにより観点単位での解析を容易にする。
