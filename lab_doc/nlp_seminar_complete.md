# 1. 自然言語処理

---

## 1-1. NLPとは

### スライドに載せる内容

自然言語処理、NLPとは：

> 人間が使う言葉を、コンピュータで扱うための技術

基本の流れ：

```text
文章
↓
コンピュータが扱える数値に変換
↓
モデルで処理
↓
分類・生成・検索・翻訳などを行う
```

例：

- ChatGPTのような対話AI
- Google翻訳
- 検索エンジン
- 文章の感情分析
- メールの自動分類
- 質問応答システム

### 話す内容

最初に強調するべきことは、コンピュータは人間のように文章をそのまま理解しているわけではない、という点である。コンピュータが扱えるのは数値なので、文章を何らかの方法で数値に変換し、その数値をモデルに入力する。

この後の2章で扱う「単語のベクトル化」は、まさにこの変換のための技術である。

### この節で扱う式

厳密な数式は不要。概念式として以下を見せるとよい。

### PowerPoint貼り付け用数式

```latex
\text{Text} \rightarrow \text{Vector} \rightarrow \text{Model} \rightarrow \text{Output}
```

### 2章との接続

1章では「文章を数値にする必要がある」と説明し、2章の「単語のベクトル化」に自然につなげる。

## 1-2. NLPの代表タスク

### スライドに載せる内容

NLPのタスクは、大きく分けると2種類ある。

### 理解系タスク

- 文章分類
- 感情分析
- 固有表現抽出
- 検索
- 文の類似度計算
- 質問応答

### 生成系タスク

- 文章生成
- 要約
- 翻訳
- 対話
- メール文作成
- コード生成

### 例

```text
文章分類：
入力「この映画は面白かった」
出力「ポジティブ」

文章生成：
入力「自然言語処理について説明して」
出力「自然言語処理とは、人間の言葉をコンピュータで扱う技術です。」
```

### 話す内容

BERTとLLMの違いを後で説明しやすくするために、ここで「理解系」と「生成系」の区別を作っておく。

BERTは主に理解系タスクで強みを持つモデルとして説明できる。一方で、LLMは生成系タスクを中心に非常に強いモデルとして説明できる。

### この節で扱う式

ここでは、入出力の形式を簡単な関数として示す程度でよい。

### PowerPoint貼り付け用数式

```latex
\text{Classification: } f_\theta(x) \rightarrow y
```

```latex
\text{Generation: } P_\theta(y \mid x)
```

### 2章との接続

分類でも生成でも、入力文はまずベクトルに変換される。そして、モデルの出力は確率として扱われる。この「ベクトル化」と「確率化」が2章の中心になる。

## 1-3. LLM・BERTの位置づけ

### スライドに載せる内容

NLPの中でのLLMとBERTの位置づけ：

```text
自然言語処理 NLP
├── 理解系モデル
│   └── BERT
│       ├── 文章分類
│       ├── 検索
│       └── 固有表現抽出
│
└── 生成系モデル
    └── LLM
        ├── 対話
        ├── 要約
        ├── 翻訳
        └── 文章生成
```

LLMとBERTの共通点：

- どちらも単語をベクトルにする
- どちらもニューラルネットワークを使う

- どちらも確率的に予測する

違い：

- LLM：次の単語を予測して文章を生成する

### 話す内容

ここではLLMとBERTを詳しく説明しない。詳しい説明は3章と4章で行う。ここでは、LLMもBERTもまったく別物ではなく、同じ共通技術の上に作られているということを伝える。

LLMとBERTの違いを、予測対象の違いとして軽く見せる。

### PowerPoint貼り付け用数式

```latex
\text{LLM: } P_\theta(w_t \mid w_1, w_2, \dots, w_{t-1})
```

```latex
\text{BERT: } P_\theta(w_m \mid \text{context})
```

## 2-1. 単語のベクトル化

### スライドに載せる内容

コンピュータは単語をそのまま扱えない。  
そのため、単語を数値ベクトルに変換する。

```text
文章
↓
トークン化
↓
単語ID
↓
Embedding
↓
単語ベクトル
```

例：

```text
「猫」 → ID 15 → [0.21, -0.43, 0.08, ...]
「犬」 → ID 32 → [0.19, -0.39, 0.11, ...]
```

Embeddingの役割：

- 単語を意味のあるベクトルに変換する
- 似た意味の単語が近いベクトルになるように学習される
- LLMでもBERTでも最初に使われる

### 話す内容

one-hot表現では、単語同士の意味的な近さを表しにくい。Embeddingでは、単語IDに対応するベクトルを学習し、意味的な特徴を持つ数値表現を作る。

初学者には、「単語を座標に置く」と説明するとよい。例えば、「犬」と「猫」は近く、「犬」と「法律」は遠い、というイメージである。

### この節で扱う式

扱うのは、**単語IDからEmbedding行列を使ってベクトルを取り出す式**である。

式の意味：

- 単語IDを指定する
- Embedding行列から対応する行を取り出す
- その単語のベクトルとして使う

### PowerPoint貼り付け用数式

```latex
x_i = E[w_i]
```

```latex
E \in \mathbb{R}^{|V| \times d}
```

```latex
x_i \in \mathbb{R}^{d}
```

### 記号の意味

- `w_i`：i番目のトークンID
- `E`：Embedding行列
- `|V|`：語彙数
- `d`：ベクトルの次元数
- `x_i`：i番目のトークンのベクトル

### LLMとの接続

LLMでは、入力文も生成途中の文章も、まずトークンに分けられ、Embeddingによってベクトル化される。

```text
今日は天気が
↓
トークンID列
↓
Embedding
↓
Transformerへ
```

### BERTとの接続

BERTでも同じく、入力文はトークン化されてEmbeddingされる。

```text
私は [MASK] を食べた
↓
トークンID列
↓
Embedding
↓
Transformer Encoderへ
```

## 2-2. ニューラルネットワークの基本式

### スライドに載せる内容

ニューラルネットワークは、入力ベクトルを別のベクトルに変換する関数である。

基本の流れ：

```text
入力ベクトル
↓
重みをかける
↓
バイアスを足す
↓
活性化関数を通す
↓
特徴ベクトル
```

ニューラルネットワークの役割：

- 入力から特徴を抽出する
- 複雑なパターンを学習する
- LLMやBERTの内部で何層も使われる

### 話す内容

ニューラルネットワークは、入力をただ記憶するのではなく、重みを使って変換しながら重要な特徴を取り出す仕組みである。

線形変換だけでは表現力が限られるため、活性化関数を入れて非線形性を持たせる。

### この節で扱う式

扱うのは、**線形変換と活性化関数の式**である。

式の意味：

- 入力に重みをかける
- バイアスを足す
- 活性化関数を通す
- 新しい特徴表現を作る

### PowerPoint貼り付け用数式

```latex
z = Wx + b
```

```latex
h = f(z) = f(Wx + b)
```

多層にする場合：

```latex
h^{(l)} = f\left(W^{(l)}h^{(l-1)} + b^{(l)}\right)
```

### 記号の意味

- `x`：入力ベクトル
- `W`：重み行列
- `b`：バイアス
- `f`：活性化関数
- `h`：出力される特徴ベクトル

### LLMとの接続

LLMのTransformer内部にはFeed Forward Networkがあり、各トークンのベクトルを変換する。

```text
Attentionで文脈を反映
↓
Feed Forward Networkで特徴を変換
↓
次の層へ
```

### BERTとの接続

BERTでもTransformer Encoder内部で同じようにFeed Forward Networkが使われる。LLMとBERTは目的が違うが、内部ではどちらもベクトル変換を大量に行っている。

## 2-3. softmaxと損失関数

### スライドに載せる内容

softmaxは、モデルの出力を確率に変換する関数である。

```text
モデルの出力値
↓
softmax
↓
各候補の確率
```

例：感情分析

```text
ポジティブ：0.85
ネガティブ：0.15
```

損失関数は、予測がどれくらい間違っているかを測る。

```text
予測確率
↓
正解と比較
↓
損失を計算
```

### 話す内容

モデルの出力は、そのままでは確率ではない。softmaxを使うことで、すべての候補の確率の合計が1になるように変換できる。

交差エントロピー損失は、正解に対して低い確率を出したときに大きくなり、正解に高い確率を出したときに小さくなる。

### この節で扱う式

扱う式は2つ。

1. **softmaxの式**  
   - 出力値を確率分布に変換する。

2. **交差エントロピー損失の式**  
   - 正解と予測のズレを測る。

### PowerPoint貼り付け用数式

softmax：

```latex
p_i = \frac{\exp(z_i)}{\sum_{j=1}^{K}\exp(z_j)}
```

交差エントロピー損失：

```latex
L = -\sum_{i=1}^{K} y_i \log p_i
```

正解クラスが1つの場合：

```latex
L = -\log p_{\text{correct}}
```

### 記号の意味

- `z_i`：クラスiに対応するモデル出力
- `p_i`：クラスiの予測確率
- `K`：候補数
- `y_i`：正解ラベル
- `L`：損失

### LLMとの接続

LLMでは、softmaxが「次に来るトークンの確率」を出すために使われる。

```text
今日は天気が
↓
softmax
↓
良い：0.50
悪い：0.20
いい：0.10
...
```

ここでの候補数は、ポジティブ・ネガティブの2クラスではなく、語彙中の全トークンである。

### BERTとの接続

BERTでは、softmaxが「[MASK]に入る単語の確率」や「分類ラベルの確率」を出すために使われる。

```text
私は [MASK] を食べた
↓
softmax
↓
りんご：0.45
パン：0.30
寿司：0.15
```

## 2-4. 勾配降下法と重み更新

### スライドに載せる内容

ニューラルネットワークは、損失が小さくなるように重みを更新する。

学習の流れ：

```text
1. 入力を入れる
2. 予測する
3. 損失を計算する
4. 勾配を求める
5. 重みを更新する
6. 繰り返す
```

キーワード：

- 損失関数
- 勾配
- 学習率
- 勾配降下法
- 誤差逆伝播

### 話す内容

損失関数で「どれくらい間違っているか」を計算した後、モデルは損失が小さくなる方向へパラメータを少しずつ動かす。

この「どちらに動かせば損失が減るか」を表すのが勾配である。勾配降下法は、損失の坂を下るようにパラメータを更新する方法だと説明するとよい。

誤差逆伝播は、出力側で計算した誤差を後ろから前へ伝え、各層の重みを更新するための仕組みである。

### この節で扱う式

扱うのは、**勾配降下法による重み更新の式**である。

式の意味：

- 現在のパラメータから、損失が増える方向の逆向きに少し動かす
- 学習率が、その動かす大きさを決める

### PowerPoint貼り付け用数式

```latex
\theta \leftarrow \theta - \eta \nabla_\theta L
```

重みに限定して書く場合：

```latex
W \leftarrow W - \eta \frac{\partial L}{\partial W}
```

連鎖律のイメージ：

```latex
\frac{\partial L}{\partial W} = \frac{\partial L}{\partial z}\frac{\partial z}{\partial W}
```

### 記号の意味

- `θ`：モデル全体のパラメータ
- `η`：学習率
- `∇_θ L`：損失の勾配
- `W`：重み

### LLMとの接続

LLMの事前学習では、次単語予測の損失を小さくするように、膨大なパラメータを更新する。

```text
次単語を予測
↓
正解トークンと比較
↓
損失を計算
↓
勾配降下法で更新
```

### BERTとの接続

BERTでは、MASKされた単語の予測や分類ラベルの予測について、損失を小さくするように重みを更新する。

```text
[MASK]を予測
↓
正解単語と比較
↓
損失を計算
↓
重み更新
```

つまり、LLMもBERTも学習の根本は同じである。

## 2-5. Attention

### スライドに載せる内容

Attentionは、文章中のどの単語に注目するかを決める仕組みである。

例：

```text
太郎は花子に本を渡した。彼は嬉しそうだった。
```

「彼」が誰を指すかを考えるには、周囲の単語との関係を見る必要がある。

Attentionの流れ：

```text
各単語をQuery, Key, Valueに変換
↓
QueryとKeyで関連度を計算
↓
softmaxで注目度に変換
↓
Valueを重み付きで集める
```

### 話す内容

Attentionは、各単語が文中の他の単語を見て、どの単語が重要かを判断する仕組みである。

Queryは「何を探しているか」、Keyは「自分がどんな情報を持っているか」、Valueは「実際に渡す情報」として説明すると分かりやすい。

### この節で扱う式

扱うのは、**Scaled Dot-Product Attentionの式**である。

式の構成要素：

- Query
- Key
- Value
- QueryとKeyの内積
- softmaxによる注目度
- Valueの重み付き和

### PowerPoint貼り付け用数式

Query, Key, Value：

```latex
Q = XW_Q, \quad K = XW_K, \quad V = XW_V
```

Attention重み：

```latex
A = \operatorname{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)
```

Attention出力：

```latex
\operatorname{Attention}(Q,K,V) = \operatorname{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
```

### 記号の意味

- `X`：入力ベクトル列
- `Q`：Query
- `K`：Key
- `V`：Value
- `d_k`：Keyの次元数
- `A`：Attention重み

### LLMとの接続

LLMでは、Attentionが過去の文脈を見るために使われる。生成時には、基本的に未来の単語を見ない。

```text
LLM：左から右に文脈を見る
```

そのため、次単語予測に適している。

### BERTとの接続

BERTでは、Attentionが前後両方の文脈を見るために使われる。

```text
BERT：前後の文脈を同時に見る
```

そのため、文章理解や穴埋め問題に強い。

## 2-6. Transformer

### スライドに載せる内容

Transformerは、Attentionを中心にしたニューラルネットワーク構造である。

基本構造：

```text
Embedding
↓
Positional Encoding
↓
Self-Attention
↓
Feed Forward Network
↓
出力
```

主な部品：

- Embedding
- Positional Encoding
- Multi-Head Attention
- Feed Forward Network
- Residual Connection
- Layer Normalization

初学者向けに特に重要な3つ：

1. Embedding  
2. Self-Attention  
3. Feed Forward Network  

### 話す内容

Transformerは、単語を順番に1つずつ処理するのではなく、文章全体の単語関係をAttentionで計算する。

ただし、Attentionだけでは単語の順番が分からないため、位置情報を単語ベクトルに加える。

Transformerには大きく分けてEncoder型とDecoder型がある。BERTはEncoder型、LLMはDecoder型を使うことが多い。

### この節で扱う式

扱う式：

- 単語ベクトルに位置情報を加える式
- Transformerブロック内のAttentionの式
- Feed Forward Networkの式
- Residual Connectionの考え方

### PowerPoint貼り付け用数式

位置情報を加えた入力：

```latex
X^{(0)} = E[w] + P
```

Transformerブロックの簡略表現：

```latex
H = \operatorname{LayerNorm}\left(X + \operatorname{MHA}(X)\right)
```

```latex
X' = \operatorname{LayerNorm}\left(H + \operatorname{FFN}(H)\right)
```

Feed Forward Network：

```latex
\operatorname{FFN}(x) = W_2 f(W_1x + b_1) + b_2
```

### 記号の意味

- `E[w]`：トークンEmbedding
- `P`：位置Embeddingまたは位置符号化
- `MHA`：Multi-Head Attention
- `FFN`：Feed Forward Network
- `LayerNorm`：層正規化

### LLMとの接続

LLMでは、TransformerのDecoder型が多く使われる。未来の単語を見ないようにマスクしながら、次の単語を予測する。

```text
Decoder型Transformer
↓
次単語予測
↓
文章生成
```

### BERTとの接続

BERTでは、TransformerのEncoder型を使う。文章全体を見て、前後の文脈から単語や文の意味を理解する。

```text
Encoder型Transformer
↓
文脈理解
↓
分類・検索・抽出
```

# 3. LLM

---

## 3-0. 全体像：Transformerで次単語を予測するモデル

### スライドに載せる内容

LLMの基本処理：

```text
入力文
↓
トークン化
↓
Embedding
↓
Transformer Decoder
↓
softmax
↓
次トークンの確率
↓
トークンを選ぶ
↓
文章に追加
↓
繰り返す
```

一言でいうと：

> LLMは、これまでの文脈から次に来るトークンを予測し続けるモデル。

### 話す内容

LLMは非常に複雑に見えるが、基本は2章で学んだ部品の組み合わせである。

- Embeddingで入力をベクトル化する
- Transformerで文脈を処理する
- softmaxで次トークンの確率を出す
- 損失関数で予測の誤差を測る
- 勾配降下法で学習する

### この節で扱う式

扱うのは、**次単語予測の条件付き確率**である。

### PowerPoint貼り付け用数式

```latex
P_\theta(w_t \mid w_1, w_2, \dots, w_{t-1})
```

短く書く場合：

```latex
P_\theta(w_t \mid w_{<t})
```

### 2章との接続

| 2章の技術 | LLMでの役割 |
|---|---|
| Embedding | 入力トークンをベクトル化する |
| Attention | 文脈中の重要な単語関係を見る |
| Transformer | 文脈を深く処理する |
| softmax | 次トークンの確率を出す |
| 損失関数 | 正しい次トークンとのズレを測る |
| 勾配降下法 | モデルを学習させる |

---

## 3-1. LLMとは

### スライドに載せる内容

LLM = Large Language Model  
日本語では「大規模言語モデル」。

特徴：

- 大量の文章データで学習する
- 非常に多くのパラメータを持つ
- Transformerを土台にしていることが多い
- 次トークン予測によって文章を生成する

得意なこと：

- 対話
- 要約
- 翻訳
- 文章生成
- 質問応答
- コード生成

### 話す内容

LLMは「人間のように理解している」と言われることもあるが、基本的には大量のテキストから言語の確率分布を学習しているモデルである。

重要なのは、LLMが自然な文章を作れる理由を「次トークン予測の積み重ね」として説明すること。

### この節で扱う式

ここでは、LLMが言語の確率分布をモデル化しているという式を扱う。

### PowerPoint貼り付け用数式

```latex
P_\theta(\text{text})
```

系列全体の確率として書く場合：

```latex
P_\theta(w_1, w_2, \dots, w_T)
```

### 2章との接続

LLMは、2章で扱った部品を大規模に組み合わせたもの。

```text
Embedding + Transformer + softmax + 損失関数 + 勾配降下法
```

---

## 3-2. 次単語予測

### スライドに載せる内容

LLMの中心は、次に来る単語を予測すること。

例：

```text
入力：今日は天気が
```

出力確率：

```text
良い：0.52
悪い：0.18
いい：0.09
晴れ：0.07
...
```

softmaxによって、語彙中の全トークンに確率を割り当てる。

### 話す内容

2章でsoftmaxを「分類確率を出す関数」として説明した。LLMでは、分類先が「次に来る可能性のある全トークン」になる。

つまり、LLMの次単語予測は、巨大な分類問題として見ることもできる。

### この節で扱う式

扱うのは、**Transformerの出力から語彙全体の確率分布を出す式**である。

### PowerPoint貼り付け用数式

```latex
p_t = \operatorname{softmax}(W_U h_t + b_U)
```

```latex
P_\theta(w_t = v \mid w_{<t}) = p_{t,v}
```

### 記号の意味

- `h_t`：時刻tの隠れ状態
- `W_U`：語彙サイズに変換する重み
- `p_t`：語彙全体への確率分布
- `v`：候補トークン

### 2章との接続

2章のsoftmaxが、ここでは「次トークン確率」を出すために使われる。分類タスクでは候補が数個だが、LLMでは候補が語彙全体になる。

---

## 3-3. 文章生成の仕組み

### スライドに載せる内容

LLMは次トークン予測を繰り返して文章を作る。

```text
今日は
↓
今日は 天気
↓
今日は 天気 が
↓
今日は 天気 が 良い
↓
今日は 天気 が 良い ので
↓
...
```

生成アルゴリズム：

1. 入力文を受け取る
2. 次トークンの確率を出す
3. トークンを1つ選ぶ
4. 入力列に追加する
5. 再び次トークンを予測する

### 話す内容

LLMが長い文章を一度に作っているように見えても、実際には1トークンずつ生成している。

トークンの選び方にはいくつか方法がある。最も確率の高いものを選ぶgreedy decoding、確率に従って選ぶsampling、ランダム性を調整するtemperatureなどがある。

初学者向けなら、まずは「確率が高い単語を順番に選んでいく」と説明すればよい。

### この節で扱う式

扱うのは、**自己回帰生成の式**である。

### PowerPoint貼り付け用数式

```latex
P_\theta(w_1, \dots, w_T) = \prod_{t=1}^{T} P_\theta(w_t \mid w_{<t})
```

Temperature付きsoftmax：

```latex
p_i = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)}
```

### 2章との接続

2章のsoftmaxは、確率を出すだけでなく、生成時にどの単語を選ぶかにも関係する。

```text
Transformer出力
↓
softmax
↓
確率分布
↓
トークン選択
```

---

## 3-4. 事前学習

### スライドに載せる内容

事前学習とは、大量の文章を使って基本的な言語能力を学習すること。

LLMの事前学習：

```text
大量のテキスト
↓
次トークン予測の問題を大量に作る
↓
正しい次トークンを予測できるように学習
```

例：

```text
入力：今日は天気が
正解：良い
```

### 話す内容

LLMは人間が一つ一つラベルを付けたデータだけで学ぶわけではない。大量の文章そのものを使い、途中までの文から次のトークンを予測する問題を作れる。

このため、非常に大量の学習データを利用できる。

### この節で扱う式

扱うのは、**次トークン予測に対する交差エントロピー損失**である。

### PowerPoint貼り付け用数式

```latex
L_{\mathrm{LM}} = -\sum_{t=1}^{T} \log P_\theta(w_t \mid w_{<t})
```

データ全体で書く場合：

```latex
L_{\mathrm{LM}} = -\sum_{(w_1,\dots,w_T) \in D}\sum_{t=1}^{T} \log P_\theta(w_t \mid w_{<t})
```

### 2章との接続

2章の交差エントロピー損失がここで使われる。

```text
予測：次トークンの確率
正解：実際に次に来たトークン
損失：正解トークンの確率が低いほど大きい
```

そして2章の勾配降下法で、損失が小さくなるように重みを更新する。

---

## 3-5. ファインチューニング

### スライドに載せる内容

ファインチューニングとは：

> 事前学習済みモデルを、特定の目的に合わせて追加学習すること。

LLMでは、指示に従うためのデータで学習することが多い。

例：

```text
入力：次の文章を要約してください
出力：要約文
```

流れ：

```text
事前学習済みLLM
↓
指示応答データで追加学習
↓
人間の指示に従いやすいLLM
```

### 話す内容

事前学習だけでは、モデルは「続きを書く」ことはできても、人間の指示にうまく従うとは限らない。

そこで、指示と望ましい回答のペアを使って追加学習する。これが教師ありファインチューニングである。

### この節で扱う式

扱うのは、**入力指示に対して望ましい出力を高確率にする損失**である。

### PowerPoint貼り付け用数式

```latex
L_{\mathrm{SFT}} = -\sum_{t=1}^{T} \log P_\theta(y_t \mid x, y_{<t})
```

### 記号の意味

- `x`：指示文・入力文
- `y_t`：望ましい回答のt番目のトークン
- `SFT`：Supervised Fine-Tuning

### 2章との接続

ファインチューニングでも、基本は2章と同じ。

```text
予測
↓
正解回答と比較
↓
損失を計算
↓
重み更新
```

違いは、学習データが「人間の指示と望ましい回答」になる点である。

---

## 3-6. 自然言語処理における強化学習

### スライドに載せる内容

強化学習とは：

> 行動の結果として得られる報酬が大きくなるように学習する方法。

基本概念：

| 強化学習の用語 | LLMでの対応 |
|---|---|
| エージェント | LLM |
| 状態 | 入力文・会話履歴 |
| 行動 | 出力する文章・トークン |
| 報酬 | 人間にとって良い回答か |
| 方策 | 出力確率分布 |

教師あり学習との違い：

```text
教師あり学習：正解データに近づける
強化学習：報酬が高い出力を増やす
```

### 話す内容

自然言語処理における強化学習は、特にLLMの調整で重要になる。

普通の次単語予測では、正しい次単語に近づける。一方、強化学習では、人間が「良い」と評価する回答に近づける。

つまり、単に文章として自然なだけでなく、役に立つ・安全・指示に従う回答を出せるようにするために使われる。

### この節で扱う式

扱うのは、**期待報酬を最大化する目的関数**である。

### PowerPoint貼り付け用数式

```latex
J(\theta) = \mathbb{E}_{y \sim \pi_\theta(y \mid x)}[R(x,y)]
```

目的を明示する場合：

```latex
\max_\theta J(\theta)
```

### 記号の意味

- `π_θ(y|x)`：入力xに対して出力yを生成する方策
- `R(x,y)`：出力yに対する報酬
- `J(θ)`：期待報酬

### 2章との接続

2章の損失関数では「正解とのズレ」を小さくした。強化学習では「報酬」を大きくする。

```text
通常学習：損失を小さくする
強化学習：報酬を大きくする
```

ただし、最終的にパラメータを調整するという意味では、2章の重み更新の考え方とつながっている。

---

## 3-7. RLHF

### スライドに載せる内容

RLHF = Reinforcement Learning from Human Feedback  
日本語では「人間のフィードバックによる強化学習」。

流れ：

```text
1. LLMが複数の回答を生成
2. 人間が良い回答を選ぶ
3. 人間の好みを学ぶ報酬モデルを作る
4. 報酬が高くなるようにLLMを調整する
```

目的：

- 指示に従いやすくする
- 役に立つ回答を増やす
- 危険・不適切な回答を減らす
- 人間にとって自然な対話に近づける

### 話す内容

RLHFは、LLMを単なる文章生成モデルから、人間にとって使いやすい対話モデルに近づけるための重要な方法である。

人間が直接すべての回答に点数をつけるのは大変なので、人間の比較データから報酬モデルを学習する。その報酬モデルを使って、LLMが高い報酬を得る出力を出しやすくする。

### この節で扱う式

扱う式は3種類ある。

1. **人間の好みを学ぶ報酬モデルの式**  
2. **報酬最大化の目的関数**  
3. **元のモデルから離れすぎないようにする制約の式**  

初学者向けには、1と2を中心に説明すればよい。

### PowerPoint貼り付け用数式

報酬モデルの比較学習：

```latex
P(y_a \succ y_b \mid x) = \sigma\left(r_\phi(x,y_a) - r_\phi(x,y_b)\right)
```

報酬最大化：

```latex
\max_\theta \mathbb{E}_{y \sim \pi_\theta(y \mid x)}[r_\phi(x,y)]
```

KL制約つき目的関数：

```latex
\max_\theta \mathbb{E}_{y \sim \pi_\theta(y \mid x)}\left[r_\phi(x,y) - \beta \operatorname{KL}(\pi_\theta \| \pi_{\mathrm{ref}})\right]
```

### 記号の意味

- `y_a ≻ y_b`：回答aが回答bより好まれる
- `r_φ(x,y)`：報酬モデルが出すスコア
- `π_θ`：調整中のLLM
- `π_ref`：元の参照モデル
- `β`：元のモデルから離れすぎないための強さ
- `KL`：分布のずれを測る量

### 2章との接続

2章の通常学習では、正解ラベルから損失を作った。RLHFでは、人間の評価から報酬を作る。

```text
通常学習：正解単語とのズレを小さくする
RLHF：人間が好む回答の報酬を大きくする
```

ただし、モデルのパラメータを更新していくという点では、2章の勾配に基づく学習とつながっている。

---

## 3-8. LLMの限界

### スライドに載せる内容

LLMの限界：

- ハルシネーション
- 最新情報に弱い場合がある
- 出力がもっともらしくても正しいとは限らない
- 計算資源が大きい
- 入力文、プロンプトに結果が左右される
- バイアスや安全性の問題がある
- 長い文脈を完全に扱えるとは限らない

原因の一つ：

> LLMは、事実を検索しているのではなく、次に来る確率の高いトークンを生成している。

### 話す内容

LLMは非常に強力だが、万能ではない。特に「自然に見える文章」と「正しい情報」は同じではない。

次単語予測に基づくモデルなので、文脈上もっともらしいが事実とは異なる文章を生成することがある。

### この節で扱う式

新しい数式は不要。次単語予測の式に戻って説明する。

### PowerPoint貼り付け用数式

```latex
P_\theta(w_t \mid w_{<t})
```

### 2章との接続

2章でsoftmaxを使って確率を出すと説明した。LLMの出力も確率に基づくため、確率的に自然な文章が、必ずしも事実として正しいとは限らない。

# 4. BERT

---

## 4-0. 全体像：Transformerで文章を理解するモデル

### スライドに載せる内容

BERTの基本処理：

```text
入力文
↓
一部の単語を [MASK] にする
↓
トークン化
↓
Embedding
↓
Transformer Encoder
↓
[MASK] に入る単語を予測
```

分類タスクの場合：

```text
入力文
↓
BERT
↓
文全体のベクトル
↓
softmax
↓
分類ラベル
```

一言でいうと：

> BERTは、前後の文脈を使って文章を理解するTransformer Encoder型モデル。

### 話す内容

BERTはLLMと同じくTransformerを使うが、目的が違う。LLMは次の単語を予測して文章を生成する。一方、BERTは文全体を見て、隠された単語や文章の意味を予測する。

BERTの理解には、2章のEmbedding、Attention、Transformer、softmax、損失関数がそのまま使える。

### この節で扱う式

扱うのは、**MASKされた単語を文脈から予測する確率**である。

### PowerPoint貼り付け用数式

```latex
P_\theta(w_m \mid w_{\setminus m})
```

### 2章との接続

| 2章の技術 | BERTでの役割 |
|---|---|
| Embedding | 入力トークンをベクトル化する |
| Attention | 前後の文脈を見る |
| Transformer | 文全体を処理する |
| softmax | MASK単語やラベルの確率を出す |
| 損失関数 | 正解とのズレを測る |
| 勾配降下法 | モデルを学習させる |

---

## 4-1. BERTとは

### スライドに載せる内容

BERT = Bidirectional Encoder Representations from Transformers

特徴：

- Transformer Encoderを使う
- 双方向に文脈を見る
- 文章理解タスクに強い
- 事前学習後にタスクごとにファインチューニングする

得意なタスク：

- 文章分類
- 感情分析
- 固有表現抽出
- 検索
- 文類似度
- 質問応答

### 話す内容

BERTの重要な特徴は「Bidirectional」、つまり双方向であること。ある単語の意味を、前の単語だけでなく後ろの単語も含めた文脈から判断できる。

これは、穴埋め問題や文章分類に向いている。

### この節で扱う式

ここでは厳密な式は不要。BERTが文全体から各トークンの表現を作ることを簡単に表す。

### PowerPoint貼り付け用数式

```latex
H = \operatorname{Encoder}(X)
```

```latex
h_i = \operatorname{BERT}(w_1, w_2, \dots, w_n)_i
```

### 記号の意味

- `X`：入力ベクトル列
- `H`：文脈化された出力ベクトル列
- `h_i`：i番目のトークンの文脈表現

### 2章との接続

BERTは、2章のTransformerのうちEncoder側を使うモデルである。

```text
BERT = Embedding + Transformer Encoder + softmax/classifier
```

---

## 4-2. Masked Language Modeling

### スライドに載せる内容

Masked Language Modeling、MLMとは：

> 文章の一部を隠し、前後の文脈から隠された単語を予測する学習方法。

例：

```text
入力：私は [MASK] を食べた。
正解：りんご
```

予測確率：

```text
りんご：0.45
パン：0.30
寿司：0.15
...
```

### 話す内容

BERTは次の単語を左から右に予測するのではなく、文の一部を隠して、その単語を当てるように学習する。

このとき、前の文脈も後ろの文脈も使える。だからBERTは文章理解に強い。

### この節で扱う式

扱うのは、**MASKされた単語の予測確率**と、**MLMの損失関数**である。

### PowerPoint貼り付け用数式

MASK単語の予測：

```latex
P_\theta(w_m \mid w_{\setminus M})
```

MLM損失：

```latex
L_{\mathrm{MLM}} = -\sum_{m \in M} \log P_\theta(w_m \mid w_{\setminus M})
```

### 記号の意味

- `M`：MASKされた位置の集合
- `m`：MASKされた位置
- `w_m`：MASK位置の正解単語
- `w_{\setminus M}`：MASKされた単語を除いた文脈

### 2章との接続

2章のsoftmaxと交差エントロピー損失がそのまま使われる。

```text
Transformer Encoderの出力
↓
softmax
↓
語彙全体の確率
↓
正解単語との損失
```

LLMとの違い：

```text
LLM：これまでの文脈から次単語を予測
BERT：前後の文脈からMASK単語を予測
```

## 4-3. 双方向の文脈理解

### スライドに載せる内容

BERTは、前後の文脈を同時に見る。

例：

```text
私は銀行でお金をおろした。
川のbankの近くを歩いた。
```

同じ単語でも、文脈によって意味が変わる。

LLMとBERTのAttentionの違い：

```text
LLM：未来の単語を見ないようにする
BERT：前後すべての単語を見られる
```

### 話す内容

2章で説明したAttentionは、文中の単語同士の関係を見る仕組みである。BERTでは、このAttentionを使って、各単語が前後のすべての単語を見られる。

そのため、文全体の意味を考慮した単語表現を作ることができる。

### この節で扱う式

扱うのは、**Self-Attentionの式と、マスクの違い**である。

BERTでは基本的に文中の前後の単語を見られる。LLMでは未来の単語を見ないようにマスクをかける。

### PowerPoint貼り付け用数式

通常のSelf-Attention：

```latex
A = \operatorname{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)
```

LLMのcausal mask付きAttention：

```latex
A = \operatorname{softmax}\left(\frac{QK^T + M_{\mathrm{causal}}}{\sqrt{d_k}}\right)
```

### 2章との接続

2章で扱ったAttentionの式は同じ。違うのは「どの単語を見てよいか」である。

```text
BERT：文全体を見る
LLM：未来を見ない
```

この違いが、BERTは理解、LLMは生成に向く理由の一つである。

## 4-4. 文章分類・検索・固有表現抽出

### スライドに載せる内容

BERTの代表的な利用例：

### 文章分類

```text
入力：この映画は面白かった
↓
BERT
↓
softmax
↓
ポジティブ
```

### 検索・文類似度

```text
文をベクトル化
↓
ベクトル同士の近さを計算
↓
似ている文を探す
```

### 固有表現抽出

```text
田中さんは東京に行った
↓
田中さん：人名
東京：地名
```

### 話す内容

BERTは、入力文から文脈を反映したベクトル表現を作る。そのベクトルを使って、分類・検索・抽出などを行う。

文章分類では、文全体を表すベクトルをsoftmaxに入れる。固有表現抽出では、各トークンごとにラベルを予測する。検索では、文ベクトル同士の近さを見る。

### この節で扱う式

扱う式は用途ごとに分ける。

1. **文章分類のsoftmax**  
2. **固有表現抽出のトークン分類**  
3. **文類似度のcos類似度**  
4. **質問応答の開始位置・終了位置予測**  

### PowerPoint貼り付け用数式

文章分類：

```latex
p = \operatorname{softmax}(W h_{\mathrm{CLS}} + b)
```

トークン分類：

```latex
p_i = \operatorname{softmax}(W h_i + b)
```

cos類似度：

```latex
\operatorname{cos}(u,v) = \frac{u^T v}{\|u\|\|v\|}
```

質問応答の開始・終了位置：

```latex
p_{\mathrm{start}} = \operatorname{softmax}(W_s H)
```

```latex
p_{\mathrm{end}} = \operatorname{softmax}(W_e H)
```

### 2章との接続

ここでも2章の技術がそのまま使われる。

- Embedding：文章をベクトル化する
- Transformer：文脈を反映する
- softmax：分類確率を出す
- 損失関数：正解ラベルとの差を測る
- 勾配降下法：タスクに合わせて更新する

## 4-5. LLMとの違い

### スライドに載せる内容

| 観点 | LLM | BERT |
|---|---|---|
| 主な目的 | 文章生成 | 文章理解 |
| 代表的な予測 | 次単語予測 | MASK単語予測 |
| 文脈の見方 | 主に左から右 | 前後両方向 |
| Transformer | Decoder系が多い | Encoder |
| 得意なタスク | 対話・要約・生成 | 分類・検索・抽出 |
| イメージ | 続きを書くモデル | 穴埋めして理解するモデル |

一言でいうと：

```text
LLM：続きを書くモデル
BERT：穴埋めして理解するモデル
```

### 話す内容

LLMとBERTは、どちらもTransformerを使う。しかし、学習目的と文脈の見方が違う。

LLMは、これまでの文脈から次の単語を予測するため、文章生成に向いている。BERTは、前後の文脈から隠された単語や文の意味を予測するため、文章理解に向いている。

### この節で扱う式

LLMとBERTの違いを、予測式で比較する。

### PowerPoint貼り付け用数式

LLM：

```latex
P_\theta(w_t \mid w_{<t})
```

BERT：

```latex
P_\theta(w_m \mid w_{\setminus M})
```

### 2章との接続

共通部品：

```text
Embedding
Attention
Transformer
softmax
損失関数
重み更新
```

違い：

```text
LLM：共通部品を文章生成に使う
BERT：共通部品を文章理解に使う
```



```latex
\theta \leftarrow \theta - \eta \nabla_\theta L
```

```latex

```



# 付録E：最終まとめスライド用文章

本授業では、自然言語処理の全体像から始め、LLMとBERTを支える共通技術としてニューラルネットワークとTransformerを確認した。単語はEmbeddingによってベクトル化され、Attentionによって文脈中の重要な関係が計算され、Transformerによって高度な言語処理が行われる。

LLMは、これまでの文脈から次のトークンを予測し、その予測を繰り返すことで文章を生成するモデルである。一方、BERTは、前後の文脈を使って隠された単語や文章の意味を予測するモデルであり、分類・検索・抽出などの理解系タスクに強い。

したがって、LLMとBERTは別々の技術として暗記するのではなく、共通の部品を異なる目的に使ったモデルとして理解するとよい。
